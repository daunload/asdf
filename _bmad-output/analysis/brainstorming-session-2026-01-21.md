## Technique Execution Results

### Assumption Reversal (가정 뒤집기)

**가정 3개:** ① 결과는 긴 글로 나와야 한다 ② 생년월일·시간·장소 입력 필수 ③ 무료 체험 + 유료 심화 구조

**뒤집기 → 차별화 후보:**

- ① 짧은 결과·카드/이모지/퀴즈형·한 화면 한 메시지 → **결과 화면 UX**
- ② 입력 최소화·단계별/선택형·"시간 모를 때" 특화 → **입력/온보딩 UX**
- ③ 과금 구조 다양화(후원, 1회성, UGC) → 비즈니스 쪽, UI/UX와는 간접

**전략적 전환:** 기능보다 **UI/UX 차별화**에 초점. ① 결과 표현, ② 입력 플로우를 UI/UX 차별화 축으로 확정.

### Cross-Pollination (다른 영역에서 옮겨오기)

**참조 서비스:** 토스, 당근마켓, 듀오링고 (심플·직관적 UI/UX)

**이식 패턴:**

| 서비스   | 결과 화면                                  | 입력·톤·지속              |
| -------- | ------------------------------------------ | ------------------------- |
| 토스     | 한 화면 한 메시지, 스텝당 한 입력, 큰 필드 | 신뢰·밝은 톤              |
| 당근     | 카드+한 줄, 따뜻·친근, 정보 최소           | 쉬운 시작, 포근한 색·형태 |
| 듀오링고 | 한 화면 한 미션, streak·가벼운 성취        | “한 번에 조금”, 응원 톤   |

**결과 1장 스펙 (합산):**

- **레이아웃:** [상단 최소] → [히어로 시각 1개] → [핵심 1~2문장] → [보조 뱃지 0~3] → [CTA 1개]. 스크롤 없음.
- **히어로:** **A-2 선택** → 심볼/아이콘 1개 (행성 ♂♀☉, 하우스, 요소 🔥 등).
- **CTA:** **B-2 선택** → `[ 다음 카드 ]` (다른 주제 한 장: 연애/일/재물/오늘 등).
- **톤:** “~해 보세요”, “~해 보는 게 좋아요” (당근·듀오). 밝고, 짧고, 말걸기.

**문구 예시 (1장형):**

- ♃ _이번 주에는 한 걸음 떨어져 보는 게 좋아요._ → `[ 다음 카드 ]`
- ♀7 _관계에서 내 감정을 좀 더 말해 보는 게 좋아요._ → `[ 다음 카드 ]`
- 🔥 _오늘은 움직이면서 생각해 보세요._ → `[ 다음 카드 ]`

**차별화 요약:** 토스+당근+듀오를 합쳐 “한 장 = 심볼 1개 + 한 줄 + 다음 카드”. 주제만 바꿔 여러 장 확장.

## Idea Organization and Prioritization

### Session Achievement Summary

- **기법:** Assumption Reversal, Cross-Pollination (2개 실행)

### Thematic Organization

**Theme 1: 결과 화면 UX**  
_한 장 구조, 스크롤 없음, 한 화면 한 메시지_

- 한 장 = 히어로(심볼 1개) + 핵심 1~2문장 + CTA 1개
- 심볼/아이콘 1개 (행성, 하우스, 요소) — A-2 선택
- `[ 다음 카드 ]` 로 주제별(연애/일/재물/오늘) 확장 — B-2 선택
- 문구 예시 3종 (♃, ♀7, 🔥) 확정
- **패턴:** 토스(한 화면 한 정보) + 당근(카드+한 줄) + 듀오(한 미션)

**Theme 2: 입력/온보딩 UX**  
_진입 장벽 낮추기, 스텝당 한 입력_

- 생년월일 → 시간 → 장소를 스텝당 하나, 큰 필드 (토스 스타일)
- "시간 모를 때" 등 최소 입력·선택형 플로우
- **패턴:** 토스의 "복잡한 것을 몇 번의 탭으로"

**Theme 3: 톤·분위기**  
_밝고, 따뜻하고, 말걸기_

- "~해 보세요", "~해 보는 게 좋아요" (당근·듀오)
- 어둡고 신비로운 대신 밝은 배경·여백·포근한 형태
- **패턴:** 당근(따뜻) + 듀오(응원) + 토스(신뢰·밝음)

**Theme 4: 전략적 방향**  
_기능 대신 UI/UX 차별화_

- 기능적 차별화는 어렵다 → UI/UX로 차별점
- 가정 3개 뒤집기: 긴 글→짧게, 필수 입력→최소/단계, (과금은 보조)
- ① 결과 표현, ② 입력 플로우를 차별화 축으로 고정

### Prioritization

| **1순위** | 결과 1장 스펙 (심볼+한줄+다음 카드) | 차별화의 핵심 시그니처, 와이어프레임·개발 기준으로 바로 사용 가능 |
| **2순위** | 톤·문구 가이드 ("~해 보세요")       | 예시 3개 이미 있음, 컨텐츠·카피 작업 시 즉시 적용                 |
| **3순위** | 입력 플로우 (스텝당 하나)           | 1장 체험 후, 입력 단계 설계·AB테스트로 이어가기 적합              |

### Action Planning

**1순위: 결과 1장 스펙**

- **이번 주:**
    - [ ] 히어로(심볼 1개)·핵심 문장·CTA 영역으로 나눈 **와이어프레임 1종** (Figma/종이)
    - [ ] 연애/일/재물/오늘 중 **주제 1개** 정해 문구 2~3개 초안
- **필요:** 디자인 도구 또는 펜/노트
- **성공:** 팀·스스로에게 "한 장이 이렇게 보인다" 공유 가능

**2순위: 톤·문구**

- **이번 주:**
    - [ ] "~해 보세요", "~해 보는 게 좋아요" **문구 가이드 1페이지** (예시 3개 포함)
    - [ ] 주제별(연애/일/재물/오늘) **추가 예시 2~3개** 작성
- **성공:** 신규 카드 문구 작성 시 이 가이드만 보고 일관성 유지

**3순위: 입력 플로우**

- **다음 단계:**
    - [ ] 생년월일 → 시간 → 장소 **스텝 수·순서** 초안
    - [ ] "시간 모를 때" **스킵·대안 플로우** 1가지 정의
- **성공:** 온보딩 플로우 1페이지로 정리, 이후 플로우/프로토타입에 반영

### Cross-cutting

- **브레이크스루:** "한 장 = 심볼+한줄+다음 카드" — 기존 "긴 글·스크롤" 가정을 깨고, 토스·당근·듀오를 한 번에 합친 포지션.
- **구현 난이도:** 결과 1장·문구는 1~2주 내 목업/프로토타입 가능; 입력 플로우는 데이터·옵션 정의 후 2~4주 수준.
